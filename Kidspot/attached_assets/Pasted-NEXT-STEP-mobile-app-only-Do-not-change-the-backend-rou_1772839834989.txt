NEXT STEP (mobile app only). Do not change the backend routes unless strictly necessary for integration.

Goal:
Connect the Expo mobile app to the existing backend API and build the initial MVP user flow.

Implement only these screens and flows:

1) Home screen
- Add a button: "Buscar lugares perto de mim"
- Request location permission
- If permission is granted, call POST /api/places/search with lat/lng
- If location is unavailable, allow fallback search by city with two quick options:
  - Franca
  - Ribeirão Preto
- Show results as a clean vertical list of cards

Each card must show:
- place name
- category (best available type)
- rating
- distance if available
- formatted address
- a simple photo if possible, otherwise a placeholder

2) Place Details screen
- Open when the user taps a place card
- Load data from GET /api/places/details?place_id=...
- Show:
  - place name
  - photo
  - address
  - rating
  - phone number if available
  - website if available
  - opening hours if available
  - button "Como chegar"
  - favorite toggle button

3) Favorites screen
- Use a fixed anonymous key for MVP, for example: "demo-user"
- Load favorites from GET /api/favorites?user_key=demo-user
- For each favorite, load place details and show a simple list
- Allow removing from favorites

4) Reviews section inside Place Details
- Load reviews from GET /api/reviews?place_id=...
- Add a simple form to submit a review:
  - rating from 1 to 5
  - 5 boolean flags:
    - trocador
    - cadeirão
    - banheiro da família
    - espaço kids
    - ambiente seguro
  - optional note
- Submit to POST /api/reviews
- Refresh review list after submit

UI requirements:
- Entire interface text must be in Brazilian Portuguese
- Clean, modern, minimal mobile-first design
- Friendly and family-oriented visual feel
- Use the KidSpot branding:
  - primary color: #FF7A00
  - secondary color: #4CAF50
- Card-based layout
- Rounded corners
- Good spacing and readable typography
- Do not implement the map yet
- Do not implement Explore or Profile yet

Technical requirements:
- Create a small API client layer for the mobile app
- Centralize backend base URL in one config file
- Handle loading, empty state, and error state on each screen
- Keep code simple and modular
- Do not add unnecessary libraries

Deliverables / acceptance criteria:
- Home screen can search places and list results
- Tapping a card opens Place Details
- Favorites can be added and removed
- Reviews can be listed and submitted
- All visible text is in pt-BR
- App runs successfully in Expo
